## 2. 工程目录规划

> **根据所选技术栈的约定目录结构。**

```
src/
├── controllers/          # 路由层：接收请求，参数校验，调用 Service
├── services/             # 业务逻辑层：核心业务规则，事务编排
├── models/               # 数据访问层：实体定义，ORM 映射
├── dto/                  # 数据传输对象：请求/响应结构体
├── config/               # 配置：数据库连接，缓存配置，环境变量
├── middleware/           # 中间件：认证，日志，异常处理，CORS
├── utils/                # 工具函数：日期格式化，加密，分页工具
├── constants/            # 常量与枚举：状态枚举，错误码定义
└── main                  # 应用入口
```

### 2.1 分层职责说明

| 层级 | 职责 | 禁止事项 |
|------|------|---------|
| Controller | 接收请求、参数绑定、调用 Service、返回响应 | 不写业务逻辑，不直接操作数据库 |
| Service | 业务规则、事务管理、多表操作编排 | 不直接处理 HTTP 细节 |
| Model/DAO | 实体定义、单表 CRUD、查询条件构建 | 不包含业务逻辑 |
